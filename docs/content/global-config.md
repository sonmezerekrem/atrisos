# Global Config

atrisos reads its global configuration from `~/.config/atrisos/config.yml`.

This file is created with defaults on first run. All fields are optional.

---

## Full schema

```yaml
# ── Stack discovery ─────────────────────────────────────────
stacks_root: "~/atrisos-stacks"   # root directory; all subdirectories are scanned as potential stacks

# extra_paths are individually registered stack dirs outside stacks_root
# managed by `atrisos register` / `atrisos unregister`, stored in registry.json
# (you don't need to edit this manually)

# ── Update behavior ─────────────────────────────────────────
update:
  default_mode: "manual"          # "manual" | "watch"
                                  # per-stack config.yml can override

# ── Traefik ─────────────────────────────────────────────────
traefik:
  acme_email: "you@example.com"   # required for Let's Encrypt TLS
  network: "atrisos_net"          # Podman network name (change only if you have a conflict)
  http_port: 80                   # host port for HTTP
  https_port: 443                 # host port for HTTPS
  image: "traefik:v3"             # Traefik image version
  dashboard:
    enabled: false                # expose Traefik dashboard via domain
    host: ""                      # e.g. "traefik.example.com" — only used if enabled: true

# ── Backup defaults ─────────────────────────────────────────
backup:
  default_destination: "~/atrisos-backups"  # used when a stack's backup.destination is not set

# ── Podman ──────────────────────────────────────────────────
podman:
  compose_command: "auto"         # "auto" | "podman compose" | "podman-compose"
                                  # auto: tries "podman compose" first, falls back to "podman-compose"
  machine_name: "atrisos"         # macOS only: name of the podman machine to use/create
  machine_cpus: 2                 # macOS only: CPUs for podman machine
  machine_memory: "2048"          # macOS only: memory in MB for podman machine

# ── Output ──────────────────────────────────────────────────
output:
  no_color: false
  no_emoji: false
  timestamp_format: "2006-01-02 15:04"   # Go time format
```

---

## Minimal config to get started

```yaml
stacks_root: "~/my-stacks"

traefik:
  acme_email: "you@example.com"
```

---

## Registry file

Extra registered paths (added via `atrisos register`) are stored separately in `~/.config/atrisos/registry.json` so they survive config file edits:

```json
{
  "extra_paths": [
    "/opt/services/monitoring",
    "/home/user/projects/myservice"
  ]
}
```

Do not edit this file by hand — use `atrisos register` and `atrisos unregister`.

---

## XDG compliance

atrisos respects `$XDG_CONFIG_HOME`. If set, it uses `$XDG_CONFIG_HOME/atrisos/` instead of `~/.config/atrisos/`.
