**Atrisos** manages Podman Compose stacks with automatic Traefik routing and TLS. Write a plain `compose.yml` with no Traefik content — atrisos injects routing labels at runtime, provisions Let's Encrypt certificates, and handles scheduling, backups, and notifications.

Targets macOS (Apple Silicon + Intel) and Ubuntu/Debian (amd64 + arm64).

---

## Install

```sh
curl -fsSL https://raw.githubusercontent.com/sonmezerekrem/atrisos/main/scripts/install.sh | sh
```

The script installs Podman if not present, configures container registry search, and places the `atrisos` binary in `/usr/local/bin` (Linux) or `/opt/homebrew/bin` (macOS Apple Silicon).

**Requirements**: macOS 13+ or Ubuntu 22.04+ / Debian 12+. Nothing else — the script handles the rest.

## Quick start

```sh
atrisos                      # first run: setup wizard (ACME email, stacks root)
atrisos init myapp           # create a stack from a template
atrisos up myapp             # start it — Traefik starts automatically
atrisos tui                  # open the interactive dashboard
```

### Stack layout

```
myapp/
├── compose.yml          # standard Compose file — no Traefik content
├── config.yml           # atrisos config: name, domains, backup, notify
├── .env                 # environment variables (auto-generated by init)
└── .env.example         # committed placeholder values
```

### config.yml

```yaml
name: "My App"

domains:
  - service: web
    host: myapp.example.com
    port: 3000
    tls: true          # true | staging | false
```

Atrisos merges Traefik labels into the compose document at runtime. Your `compose.yml` is never modified.

## Templates

Run `atrisos init` to create a stack from a built-in template. Passwords and domains are auto-generated — press Enter to accept or type your own value.

| Template | Description |
|---|---|
| `basic` | Single service with optional domain routing |
| `webapp-postgres` | Web service + PostgreSQL with backup |
| `postgres` | PostgreSQL 16 Alpine |
| `valkey` | Valkey 8.1 (Redis-compatible) |
| `mongo` | MongoDB 8 |
| `wordpress` | WordPress + MySQL 8 with TLS |
| `registry` | Private container image registry |

## Commands

```sh
# Stack lifecycle
atrisos up <stack>           # start (Traefik auto-starts on first use)
atrisos down <stack>         # stop
atrisos restart <stack>      # restart
atrisos update <stack>       # pull latest images and recreate
atrisos up --all             # operate on all stacks
atrisos up --tag production  # operate on stacks by tag

# Inspect
atrisos status               # status of all stacks
atrisos logs <stack>         # tail logs
atrisos ps <stack>           # list containers
atrisos render <stack>       # print merged compose YAML (debug routing)
atrisos validate <stack>     # dry-run config and compose validation
atrisos outdated             # check for newer image versions

# Maintenance
atrisos backup <stack>       # manual restic backup of named volumes
atrisos export <stack>       # export stack as .tar.gz (no .env)
atrisos import <file>        # import stack archive
atrisos self-update          # update atrisos binary in place

# Traefik
atrisos traefik status       # Traefik container status
atrisos traefik logs         # tail Traefik logs
atrisos traefik restart      # restart Traefik
```

## How it works

```
Internet
   |
Traefik        <- managed at ~/.config/atrisos/traefik/
   |
atrisos_net    <- shared Podman network
   |--------- myapp-web    Host(`myapp.example.com`) :3000
   |--------- blog-ghost   Host(`blog.example.com`)  :2368
   |--------- myapp-db     (no domain entry — not exposed)
```

1. User runs `atrisos up myapp`
2. atrisos reads `compose.yml` + `config.yml`, injects Traefik labels and network into routed services in memory, writes to a temp file
3. `podman compose -f <tmpfile> up -d` is called — the original `compose.yml` is never touched
4. Traefik picks up the new container labels immediately; Let's Encrypt issues a cert on the first HTTPS request

## Build from source

```sh

cd atrisos
make build                       # ./atrisos
make install                     # /usr/local/bin/atrisos
cd app && go test ./internal/... # run tests
```

Go 1.26+ required.
